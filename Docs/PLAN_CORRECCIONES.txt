ETAPA 3 — RBAC: ACTIVAR EL MIDDLEWARE DE ROLES  [BACKEND]
--------------------------------------------------------------------------------
La funcion authorize() ya existe y funciona, pero nunca se aplica en las rutas.
Cualquier usuario puede hacer llamadas de admin. Facil de corregir.

  [x] BUG-B01  authMiddleware.ts — authorize() sin usar en ninguna ruta
               Proteger las rutas que requieren rol admin:
               - POST /api/rooms (crear habitacion) → solo admin
               - PUT/PATCH /api/rooms/:id → solo admin
               - DELETE /api/rooms/:id → solo admin
               - GET /api/reservations/admin/all → solo admin

  Archivos:  backend/src/routes/roomRoutes.ts
             backend/src/routes/reservationRoutes.ts
             backend/src/middlewares/authMiddleware.ts

ETAPA 4 — QUERY JOIN INCORRECTO EN RESERVAS  [BACKEND]
--------------------------------------------------------------------------------
El join de la query de reservaciones tiene un alias incorrecto que puede
causar que los datos de tipo de habitacion no lleguen bien al frontend.

  [ ] BUG-B02  reservationController.ts linea 8 — join mal formado
               Cambiar:
                 .select('*, rooms(*), room_types:rooms(room_types(*))')
               Por:
                 .select('*, rooms(*, room_types(*))')

  Archivos:  backend/src/controllers/reservationController.ts
